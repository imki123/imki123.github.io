2.OAuth 로그인 적용 방법 (구글 로그인, 네이버 로그인, 카카오 로그인)
194, 2020-10-12
안녕하세요 임기입니다 😄 이번 포스팅 주제는 OAuth입니다. OAuth(Open Authorization)는 열린 인증으로 공용로그인 등에 사용됩니다.  다들 네이버, 카카오, 구글 아이디 등으로 한 번씩 로그인 해보셨죠? OAuth는 이미 만들어져있는 계정을 활용해서 새로운 계정을 만들거나 로그인을 가능하게 해주는 서비스랍니다. 제 블로그도 이번에 네이버, 카카오, 구글 로그인 OAuth를 적용했답니다~ 이제 더 편리하게 댓글을 달 수 있어요 ㅎㅎㅎ [로그인하러 가기] 카카오를 예로 들어서 웹 OAuth 적용 방법을 알려드릴게요~ 1. OAuth는 해당 서비스의 SDK(Software Development Kit)를 이용해서 사용자 인증, 로그인, 로그아웃 등의 기능을 사용할 수 있어요. 가장 먼저 SDK를 불러와서 <head>에 적용해주세요. <!--index.html --> <!DOCTYPE html> <html>   <head>     <!-- 카카오 로그인 스크립트 -->     <script src="https://developers.kakao.com/sdk/js/kakao.min.js"></script> <!-- or 아래처럼 라이브러리를 다운로드 받아 프로젝트에 포함시키는게 로딩 속도가 더 빠르겠죠? -->     <script src="./lib/kakao.min.js"></script> </head> ... </html> CDN으로 로딩 시마다 카카오 사이트에서 직접 받아오셔도 되고 저처럼 미리 다운로드 받아서 프로젝트에 포함하셔도 돼요. 저는 로딩속도 향상을 위해 프로젝트에 포함시켜놨어요 ㅎㅎ 2. SDK를 이용해서 인증 객체를 초기화 해줍니다. 인증 객체를 초기화하기 위해서는 클라이언트 아이디가 필요한데요. 이건 해당 서비스로 접속해서 애플리케이션을 생성하면 된답니다. 카카오의 경우 카카오디벨로퍼 사이트에서 애플리케이션을 생성하시고 발급받은 Javascript 키(클라이언트 아이디)를 사용하시면 돼요. [카카오 디벨로퍼스 링크] 또 callbackUrl이나 redirectUrl을 설정해야 하는데 로그인 정보를 사용할 페이지들을 적어주시면 돼요. 제 블로그 같은 경우는 https://imki123.github.io/login 이랍니다. 이제 SDK로 인증 객체를 초기화 해볼게요. Kakao.init()으로 인증 객체를 초기화하고, Kakao.isInitialized()로 초기화 여부를 판단할 수 있어요. <body> <!-- 카카오 로그인 초기화 Script -->     <script>       // SDK를 초기화 합니다. 사용할 앱의 JavaScript 키를 설정해 주세요.       Kakao.init('0d74caaceac5f0f14c0d6a667a9dbdee')       // SDK 초기화 여부를 판단합니다.       console.log(Kakao.isInitialized()) <script> ... </body> 3. 인증 객체를 이용해서 로그인, 로그아웃 함수를 만들어줍니다. <body> ... <script> function kakaoLogin() {   //로그인하고   Kakao.Auth.login({     success: function (response) {       //사용자 정보 가져오기       Kakao.API.request({         url: '/v2/user/me', //계정 정보를 가져오는 request url         success: function (response) {           let user = response.kakao_account //카카오 계정 정보           //console.log(user)           user.host = 'kakao' //다른 로그인 서비스와 구분하기 위해서 개인적으로 추가했음 // 해당 페이지에서 객체를 만들고 곧바로 user 정보를 사용할 수 도 있고, // input 엘리먼트에 json으로 저장해뒀다가 나중에 사용할 수도 있음. 여기서는 input에 저장           const userinfo = document.querySelector('#userinfo')           if (userinfo) userinfo.value = JSON.stringify(user) //user를 json문자열로 변환해서 저장해두기         },         fail: function (error) {           console.log(error)         },       })     },     fail: function (error) {       console.log(error)     },   }) } function kakaoLogout() {   if (Kakao.Auth.getAccessToken()) {     //토큰이 있으면     Kakao.API.request({       //로그아웃하고       url: '/v1/user/unlink',       success: function (response) {         //console.log(response)       },       fail: function (error) {         console.log(error)       },     })     //토큰도 삭제     Kakao.Auth.setAccessToken(undefined)   } } </script> </body> 로그인은 Kakao.Auth.login(), 사용자 정보는 Kakao.API.request({url: '/v2/user/me'}, 로그아웃은 Kakao.API.request({url: '/v1/user/unlink'}), 토큰(로그인 정보) 삭제는 Kakao.Auth.setAccessToken()을 사용합니다. 더 자세한 API는 [카카오 디벨로퍼스]에서 확인하면 됩니다. 4. 로그인, 로그아웃 버튼을 만들고 함수를 적용합니다. <body> ... <button id="kakaoLogin" onclick="kakaoLogin()">kakaoLogin</button>   <button id="kakaoLogout" onclick="kakaoLogout()">kakaoLogout</button> <input id="userinfo" value="" /> <!-- 유저정보를 저장할 input 엘리먼트 --> </body> 저는 input을 만들고 유저정보를 저장한 다음, 필요시에 input 엘리먼트에서 꺼내 사용했어요. 이건 상황에 맞춰서 선택하시면 돼요. 5. 사용자 정보를 저장하고, 필요시에 백엔드로 보내 토큰을 만들고 쿠키에 저장하면 끝! <body> ... <script> let user const userinfoElem = document.querySelector('#userinfo') //유저정보가 들어있는 input 엘리먼트 //1초마다 유저정보가 있는지 체크하기 let frame = setInterval(function(){ console.log('유저정보 체크')       if (userinfoElem && userinfoElem.value) { //유저정보가 있으면 체크 멈추고 clearInterval(frame) //유저 정보를 저장 user = JSON.parse(userinfoElem.value) //필요하면 백엔드로 유저정보 보내서 토큰 만들고 쿠키에 저장하기 Axios.post('backend_url', {user: user}, {withCredentials: true}) .then(res => { console.log(`${user.username}로그인 성공`) }) .catch(error => { console.log(error) }) } }, 1000) </script> </body> 자 여기까지 OAuth를 적용하는 방법을 알아봤습니다~ OAuth를 적용하기가 마냥 쉽지 않지만, 개발자가 고생할 수록 사용자들이 더 편리한 서비스를 사용할 수 있다는 걸 명심하고 많이 노력해야합니다 ㅎㅎㅎ 😄 포스팅은 여기서 1.JS에서 this란?
38, 2020-09-09
1. 기본적으로 this는 window다. function 으로 생성하든 콜백함수든 어디서든 window다. console.log(this) //window function(){ console.log(this) //window } 2. 객체메서드, new 생성자, 화살표 함수, bind, call, apply 에서는 바뀐다. //객체메서드 const counter = { value : 0 inc : function(){ // 여기서 this는 counter이다 this.value ++ } } //new 생성자 function Person(){ this.name = '기영' } Person.prototype.hello = function(){ console.log(this.name) //여기서 this는 Person의 인스턴스이다. } const person = new Person() console.log(person.name) //기영 //call, apply, bind a.call(obj, 1, 2, 3) //a로 obj의 메서드를 사용한다. a.apply(obj, [1,2,3]) //a로 obj의 메서드를 사용한다. a = obj.bind(obj2) //obj2의 메서드를 a에 바인딩한다. a() 3. 예외적으로 onclick 등의 이벤트리스너나 jquery등의 라이브러리에서도 바뀔수있다. button.onclick = function(){ console.log(this) //여기서 this는 button }
