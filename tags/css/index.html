2.CSS 꿀팁
2020-09-20
안녕하세요 임기입니다 :D CSS 꿀팁을 몇 가지 적어보고자 글을 씁니다. 저는 프로젝트를 시작하게 되면 가장 먼저 추가하는 스타일이 있는데요 바로 box-sizing 스타일입니다. * { box-sizing: border-box; } 박스사이징을 설명하기 전에 일반적인 상황을 먼저 보여드릴게요. 박스사이징을 설정하지 않은 상태에서 width : 100px인 버튼을 만들고 싶다면 아래와 같이 설정하게 됩니다. button { width: 100px; padding: 8px; border: 1px solid black; } 그런데 이렇게 되어있다면 실제로 button의 width는 100 + 8*2 + 1*2 = 118px이 됩니다. 저희가 원하는 크기는 100px인데 훨씬 커지므로 레이아웃이 깨져서 width를 82px로 줄여줘야하는 불편함이 있죠 :( 이런 불편함을 해결할 수 있는게 box-sizing 스타일입니다. 박스사이징을 설정하게 되면 border, padding, content(width, height) 요소가 content 사이즈를 넘어가지 않게 됩니다. 즉, width: 100px을 설정하면 자동으로 border와 padding을 감안해서 width가 82px인 것 처럼 보이게 되는 것이죠. 박스사이징은 width: 100% 처럼 퍼센티지 요소를 적용할 경우도 아주 잘 작동하게 됩니다. 단, 조심할 점이 하나 있는데요. margin은 width 밖에 적용되므로 margin을 적용해야하는 경우는 부모 엘리먼트의 전체 너비를 넘지 않도록 사이즈 조정을 해주어야 레이아웃이 깨지지 않습니다. 저는 이럴 때 CSS 함수인 calc()를 주로 사용합니다. 예를 들어서 menu와 content를 좌우로 배치하는 경우를 생각해봅시다. <!-- example.html --> <body> <div id="menu"/> <div id="content"/> </body> /* example.css */ #menu{ width: 300px; } #content{ width: 100%; } 위와 같이 스타일을 설정하면 menu와 content의 너비의 합이 100% + 300px이 되어서 레이아웃이 깨지게 됩니다. 그래서 이 두개를 같이 정렬하기 위해서는 width를 계산해서 입력해줘야합니다. 아래와 같이 말이죠. /* example.css */ #menu{ width: 300px; } #content{ width: calc(100% - 300px); } 주의 해야할 점은 calc() 안에 사칙연산 좌우로 띄어쓰기를 해주어야 잘 작동합니다. 띄어쓰기 없이 붙여쓰게 되면 CSS가 동작하질 않아요. 세번 째 꿀팁은 display: flex 입니다 :D flex는 요소들을 좌우 또는 상하 배치할 때 요긴하게 사용됩니다. 아래 예시를 볼까요? <!-- example.html --> <div class="flexDiv"> <button>1번</button> <button>2번</button> <button>3번</button> <button>4번</button> </div> /* flex.css */ .flexDiv{ background: lightblue; width: 100%; height: 100vh; display: flex; justify-content: space-between; align-items: flex-start; } button{ background: green; color: white; } 위 스타일을 적용하면 아래 그림 처럼 버튼이 배치되게 됩니다. .flexDiv 안에 버튼들이 나란히 위치해 있죠? 이걸 가능하게 해주는 속성이 justify-content입니다. justify-content에는 몇 가지 값들이 있는데 이걸 알아두셔야 상황에 맞는 스타일을 사용하실 수 있습니다. justify-content는 기본 축(default: row) 방향으로 요소들을 배치합니다. 기본 축이 가로이므로 한 줄에 여러개 요소들이 배치되는 모습이죠. space-between : 좌우 끝에 요소를 하나 씩 배치하고 남은 요소들을 가운데에 균일하게 배치합니다. 보통 페이지의 헤더 부분에 많이 사용하는 방식이죠. 위에 있는 그림처럼 배치됩니다. space-around : 모든 요소를 균일하게 배치합니다. 아래와 같은 모양이죠 flex-start : 축의 시작(왼쪽)에 요소들을 배치합니다. 딱히 flex를 쓸 필요가 없겠죠? flex-end : 축의 끝(오른쪽)에 요소들을 배치합니다. center : 축의 가운데에 요소들을 배치합니다. 다음으로 알아볼 속성은 align-items입니다. align-items는 justify-content와 반대 축을 기본 축으로 가집니다. justify-content가 가로 축이었다면 align-items는 세로 축이 되겠죠? 이번에는 버튼의 높이를 다르게 주어서 테스트 해봅시다. flex-start는 위쪽으로 정렬됩니다. 아래 이미지와 같습니다. flex-end는 아래쪽으로 정렬됩니다. 아래 이미지와 같습니다. center는 중앙으로 정렬됩니다. 저는 center나 flex-end를 가장 많이 사용합니다. 글과 이미지를 같은 높이로 맞추고 싶을 때 center가 유용하게 사용됩니다. 아래 이미지와 같습니다. 어떤가요 요소들을 편리하게 배치할 수 있겠죠? flex는 이외에도 축 방향 설정, 줄바꿈 등 몇 가지 설정을 더 할 수 있습니다. 더 고급 정보는 추후에 포스트할게요~ 이상입니다 :D
1.CSS 애니메이션에 대해 알아보자
2020-09-21
안녕하세요 임기입니다 :D CSS 애니메이션에 대해서 알아볼건데요. 간단한 사용 예시는 아래와 같습니다. /* animation.css */ #refresh { width: 48px; height: 48px; } .refresh { animation: refreshing 1s linear infinite; } @keyframes refreshing { 0% { transform: rotate(0deg); } 100% { transform: rotate(360deg); } } 사용방법은 정말 간단합니다. @keyframes 뒤에 애니메이션 이름을 적어줍니다. 여기서는 refreshing입니다. 그리고 프레임 마다 어떤 동작을 할지 적어주시면 됩니다. 프레임은 0% ~ 100% 사이의 퍼센티지를 작성해주시면 프레임 간에 연속된 애니메이션이 동작됩니다. 이제 애니메이션을 언제 시작하고 종료할지 고민하셔야합니다. 예시는 refresh이니 새로고침 버튼을 만들어서 버튼을 누르면 애니메이션이 동작하고, 버튼을 다시 누르면 애니메이션이 종료되게 해봅시다. 저는 주로 querySelector를 이용해서 classList를 추가하고 제거하는 방법을 사용합니다. <!-- example.html --> <body> <img src="./refresh.png" id="refresh"/> <div> <button onClick="refreshing()">새로고침</button> </div> <script src="./example.js"></script> </body> // example.js let refreshToggle = false const refresh = document.querySelector("#refresh") function refreshing() { if (refresh) {     if (!refreshToggle) { // toggle이 false이면 클래스 추가     refresh.classList.add("refresh")     refreshToggle = true     } else { // toggle이 true이면 클래스 제거     refresh.classList.remove("refresh")     refreshToggle = false     } } } 새로고침 버튼을 클릭하면 refresh 클래스가 추가되면서 애니메이션이 동작하고, 버튼을 한 번 더 클릭하면 클래스가 삭제되면서 애니메이션이 사라지는 예시입니다. 이제 animation 속성에 대해서 좀더 알아볼건데요. animation에는 animation-name, delay, direction, duration, fill-mode, iteration-count 속성 등이 있습니다. name: @keyframes에서 만든 애니메이션 이름 delay: 애니메이션이 시작하기 전 지연 시간. default) 0s direction: 애니메이션의 동작 방향. 정방향은 normal, 역방향은 reverse, alternate는 매 사이클마다 방향을 전환. duration: 애니메이션이 한번 반복되는 시간. ex) 1s fill-mode: 애니메이션이 끝날 때의 스타일 지속여부. none, 스타일 지속하고 싶으면 normal. default) none iteration-count: 애니메이션 반복 회수. ex) 1, 1.5, infinite 저는 보통 한 줄에 표현하는 걸 선호합니다. animation: refreshing 1s linear infinite; 이것은 refreshing 애니메이션을 1초 동안 linear하게 무한히 반복하겠다. 라는 뜻이랍니다. 여기서 linear는 애니메이션의 속도를 조절하는 속성인데요. linear는 일정하게, ease는 부드럽게 라고 생각하시면 됩니다. 일정하게 회전해야 하는 경우는 보통 linear를 사용하구요. 위치나 크기가 변경되는건 ease를 사용하면 부드럽게 표현된답니다. 다음엔 CSS transition에 대해서 알아볼까해요~ 애니메이션과 비슷하지만 간단하게 설정할 수 있어서 자주 사용한답니다. 이상입니다 :D 참고: https://developer.mozilla.org/ko/docs/Web/CSS/CSS_Animations/Using_CSS_animations
