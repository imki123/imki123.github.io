GitHub Pages 설정해보기

안녕하세요 임기입니다 :D

오늘은 GitHub Pages에 대해 알아봅시다.


GitHub Pages는 github에서 제공하는 무료 웹호스팅 서비스입니다.

우리가 github에 html파일을 저장하면 인터넷을 통해 웹페이지를 볼 수 있다는 거죠. 그것도 무료로 말이에요. 대단하죠?


매우 쉬운 사용방법을 알려드릴게요.

우선 github에 가입하고 나서 레포지토리를 만들고 Readme.md 파일을 자동 생성하고 Settings에 들어가봅시다. 그러면 스크롤 중간쯤에 GitHub Pages라는 섹션이 있을겁니다. 여기서 Source의 Branch를 선택해주면 설정이 끝납니다. 기본으로 master 브랜치를 설정해주시면 해당 브랜치의 html파일 또는 md파일을 웹 화면에 보여주게됩니다. 이렇게 간단하게 정적 html을 이용해서 웹페이지를 만들 수 있습니다. 대부분의 github.io 블로그들이 해당되죠.


그외에도 동적인 방법을 사용할 수도 있는데요. React 같은 SPA(Single Page Application)가 이에 해당합니다. SPA는 여러개의 html페이지로 나뉘어 있는게 아니라 JS를 이용해서 웹페이지를 렌더링하는데요. 이 때문에 해당 URL로 직접 접근하려하면 404 오류를 내며 페이지를 찾을 수 없다는 페이지가 나타날 겁니다. 이런 SPA도 웹호스팅을 가능하게 해주는게 gh-pages라는 node 패키지입니다.

gh-pages는 바벨이라는 웹 빌더로 압축된 html 파일과 js 파일을 빌드합니다. 그리고 디플로이를 하게 되면 빌드된 파일을 gh-pages 브랜치에 push하게 됩니다. 이 때 github의 Settings에서 Source를 gh-pages 브랜치로 선택해주면 빌드된 SPA를 확인할 수 있답니다.


기본적으로 호스팅 주소는 https://[username].github.io/[repository name] 으로 설정됩니다.

여기서 꿀팁은 repository name을 생략할 수 있는데요. repository name을 [username].github.io로 설정하면 호스팅 주소는 https://[username].github.io/ 가 됩니다. 이렇게 하면 도메인 뒤의 파라미터를 생략할 수 있으므로 여러가지 설정에서 이점을 볼 수 있답니다. 아래 이미지처럼 깔끔한 도메인을 만들 수 있어요.


이 블로그도 이와 같은 방식으로 제작되서 호스팅되고 있답니다 :D

여러분들도 손쉽게 웹호스팅을 시작해보세요~
