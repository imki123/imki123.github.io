귀여운 React Native 앱 Catbook!

안냥하세요 임기입니다 :D

오늘은 React Native로 만든 Catbook을 소개하려고 글을 썼어요.

Catbook은 귀여운 고양이와 강아지 이미지를 볼 수 있는 간단한 네이티브 앱이랍니다.

https://imki123.github.io/catbook/



그런데 React Native가 뭐냐고요?


React Native는 facebook에서 제작한 react와 유사한 방식으로 모바일 native앱을 만드는 프론트엔드 자바스크립크 라이브러리랍니다. react 코드를 작성할 줄 알면 아주 쉽게 RN 앱도 만들 수 있어요

RN의 가장 큰 장점은 크로스플랫폼을 지원한다는 겁니다. 하나의 코드로 안드로이드, ios, web 앱을 동시에 만들수 있어요! 정말 놀랍죠? RN를 사용하지 않으면 똑같이 동작하는 프로그램을 만들기 위해서 세가지 언어로 세번 코딩을 해야하는 끔찍한 일이 벌어진답니다.


RN는 이 외에도 정말 많은 장점이 있답니다 :D

RN에는 정말 편리한 기능인 hot-reloading이 있어요. 네이티브 앱의 경우 코드를 수정하면 코드를 새로 컴파일하고 그걸 에뮬레이터에 올리고 테스팅 하는데 시간이 꽤 걸려요. 그래서 개발을 하다보면 흐름이 끊겨 답답하고 컴파일 하는데만 수 시간을 소모하곤 한답니다. 하지만 hot-reloading은 state를 초기화하지 않고 수정한 코드를 바로 화면에 적용시켜주어서 테스팅 시간을 획기적으로 줄일 수 있답니다.

live-reloading은 소스코드 수정 시 앱 전체를 새로고침 하는건데요. 설정을 통해서 선택해주시면 됩니다.


그리고 RN을 얘기하다보면 빼놓을 수없는게 있는데요. 바로 expo입니다.

RN을 처음 세팅하려면 꽤나 어렵고 복잡한 설정들을 잡아줘야합니다. 웹개발자에게 이런 세팅은 상당히 번거로운 일이죠. 이를 간편하게 세팅해주는게 바로 expo입니다.

expo는 이외에도 아주 강력한 기능을 지원해줍니다. 바로 expo app을 통해 내 스마트폰에서 실시간으로 디버깅을 할 수 있는건데요. 정말 놀라운건 윈도우 환경에서 만든 앱을 바로 아이폰에서 테스팅 할 수 있습니다.. 언빌리버블.. 정말 expo 팀에게 찬사를 아끼지 않을 수 없네요.


또 아주 훌륭한 기능이 있는데요. 바로 expo publish입니다. 네이티브 앱은 구글플레이나 앱스토어에 앱을 게시해야 다른 사람들이 앱을 설치하고 사용할 수 있는데요. 앱을 게시하고 적용되는데 꽤 시간이 소요됩니다. 길게는 일주일 이상 수정사항을 적용하지 못하는 경우가 생길 수 도 있습니다. 자그마한 수정사항이라도 내가 원할 때에 앱을 게시 할 수 없다는건 아주 큰 리스크가 될 수 있죠.


하지만 expo는 실시간 publishing을 지원합니다! 내가 앱을 수정하고 expo publish를 하면 수정사항이 곧바로 앱에 적용됩니다. 정말 놀랍지 않을 수가 없습니다...

여기서 끝이 아닙니다. expo는 RN에서 다루지 않는 다양한 기능들을 지원합니다. 예를 들어 gps의 경우 RN에서 제공하는 기능은 몇 가지 뿐이지만 expo는 수 십 가지의 기능을 지원합니다. expo를 활용하시면 개발에 큰 이점을 줄 수 있답니다.


여러분들도 expo를 이용해서 간단한 네이티브 앱을 만들어보시면 어떨까요 ^^?
