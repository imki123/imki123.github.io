CORS 설정 체험기 (Koa, fetch, axios)

cors는 어렵다.

cors는 Cross-Origin-Resource-Sharing의 준말이다.

즉, 서로 다른 출처 간의 자원 공유를 차단하기 위한 정책이다.

어떤 웹페이지에서 다른 도메인의 자원을 가져오고자 할때, 도메인이 다를 경우 이를 차단하는 정책이다.


나같은 경우는 프론트에서 백엔드 정보를 가져와야하는데 둘의 도메인이 달라서 설정해줬다.

cors는 두군데에서 설정해야한다. 백엔드, 프론트엔드 둘다이다.



백엔드는 

1. 서버프레임워크에 cors 정책을 설정한다.

const app = Koa()
app.cors({
  origin: 'https:kiyoung.com', 
  credentials: true, 
  exposeHeaders: ['page-count']
})

등을 설정해야한다.


2. 서버의 https 프로토콜을 설정한다.

app.proxy = true

koa의 경우 proxy를 설정해줘야 https 프로토콜이 동작한다.


3. 쿠키를 set 하기위해서는 서버의 cookieOptions도 설정해줘야한다.

const cookieOptions = {
  secure: true, //https 프로토콜
  sameSite: 'none'
}

로컬 환경에서 테스트 할때는 http 프로토콜이므로 secure를 false로 바꿔주어야한다. 안하면 error가 발생한다..

로컬 환경에서는 ctx.request.protocol을 조회해서 간단한 메서드를 만들어 미들웨어에 추가해주자.



프론트엔드는 데이터를 요청할 때 credentials 설정을 포함하여 전송해야한다.

1. axios의 경우

axios.post(url, {
  withCredentials: true
})

를 설정해야한다.


1. fetch의 경우

fetch(url,{
  method: 'POST',
  mode: 'cors',
  credentials: 'include'
})


p.s) axios 라이브러리로 cookie-set을 할 경우 잘 안된다. 아무래도 axios의 오류인 것으로 생각된다. 기본 자바스크립트 라이브러리인 fetch는 잘 작동한다. 그래서 대부분 axios를 사용하고 ctx.cookie나 ctx.state를 조작해야 하는 경우는 fetch를 사용하고 있다. 둘 다 사용하다보니 둘 다 까먹지 않아서 좋은 것 같기도 하다. axios를 써서 편하게 하는 것도 좋지만 fetch도 사용할 줄 알아야 하겠지?
