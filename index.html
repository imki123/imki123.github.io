1.비동기함수를 반복문에서 사용하는 방법(setTimeout, for, promise, setInterval, 실행컨텍스트)
안녕하세요. 임기입니다. 😄 오늘은 자바스크립트에서 흔하게 발생하는 비동기함수를 반복문에서 사용하는 방법을 알아볼거에요. 비동기함수는 대표적으로 타임함수(setTimeout, setInterval)가 있고, fetch 등의 AJAX 함수들, 또는 Promise를 리턴하는 함수들이 있겠습니다. 그러면 반복문과 비동기 함수를 같이 사용하는 방법을 알아봅시다. 대표적인 반복문인 for로 예시를 들어볼게요. //for문 안에서 setTimeout() 함수 사용하기 for(var i=0; i<4; i++){   setTimeout(() => {    console.log(i) // 어떤 값이 출력될까요???   },1000) }​ 자, for가 0~3까지 4번 실행되는 반복문을 만들고, 그 안에 setTimeout함수를 이용해서 1초 후에 i값이 콘솔에 출력되도록 하는 코드입니다. 과연 의도대로 출력이 될까요?? 만약 그랬다면 문제로 내지 않았겠죠? ㅎㅎ 저희가 원한 결과는 1초 마다 0, 1, 2, 3을 한번씩 출력하는 것입니다. 하지만 실제 결과는 1초 후에 4를 거의동시에 4번 출력하게 됩니다. 의도와 상당히 다른 결과가 나와버렸죠? 이제 이 문제를 해결해봅시다. 이 문제를 풀기 위해서는 실행컨텍스트와 스코프, 호이스팅 개념을 알고 있어야합니다. 추가적으로 클로저와 즉시실행함수, 콜스택, 이벤트루프 까지 알고 있으면 더 좋습니다. 한번에 다 다루기엔 좀 어렵죠? 😭 우선 실행컨텍스트에 대해서 알아봅시다. 자바스크립트엔진은 스크립트를 실행하다가 함수를 만나게되면 실행컨텍스트라는 것을 생성하게 되는데요. 이 실행컨텍스트에 함수가 실행될 때의 환경을 저장하게 됩니다. 이렇게 만들어진 실행컨텍스트는 자신만의 스코프(유효범위)를 갖게됩니다.예를 들어볼까요? var x = 10 var y = 20 function print(){ var x = 5 console.log(x, y) } print() 자 위의 결과는 어떻게 될까요? 어렵지 않게 5와 20이 출력된다는걸 알 수 있습니다. 그러면 실행컨텍스트 입장에서 살펴봅시다. GO( x=10, y=20, print = function ) print 컨텍스트( x=5 ) 참조한 변수 : x=5, y=20 우선 자바스크립트가 실행되면 글로벌 오브젝트(GO)를 생성하게됩니다. GO에는 전역 변수들과 함수들이 프로퍼티로 저장되게 됩니다. 여기서는 x = 10, y = 20, print = function 정도가 저장되어 있겠네요. 자바스크립트엔진이 print() 함수를 실행하게되면 새로운 실행컨텍스트가 생성됩니다. print 컨텍스트는 x = 5를 프로퍼티로 저장합니다. 그렇다면 y는 어디서 찾을 수 있을까요? print 컨텍스트에 y라는 변수는 들어있지 않으므로 바로 상위 컨텍스트에서 y를 찾게 됩니다. 다행히 y가 GO에 저장되어 있어서 y가 20인 것을 알 수 있겠네요. 만약에 GO에도 y가 선언되어있지 않다면 'y is not defined' 라는 컨퍼런스 에러를 발생시킵니다. 즉, 변수를 참조할 땐 함수가 실행된 컨텍스트 내부를 먼저 살펴보고 없으면 상위 컨텍스트를 계속해서 찾게됩니다. 이제 for문 예시를 실행컨텍스트 입장에서 살펴봅시다. //for문 안에서 setTimeout() 함수 사용하기 for(var i=0; i<4; i++){   setTimeout(() => {    console.log(i) // 어떤 값이 출력될까요???   },1000) }​ GO( i=0 ) for 컨텍스트 ( empty ) setTimeout 컨텍스트 ( empty ) for문 안의 var i는 전역변수로 호이스팅됩니다. 그리고 for문이 시작되면 0으로 할당되죠. 그 다음 곧바로 setTimeout 함수를 만나게 되죠. 문제는 setTimeout 함수는 1초가 지날 때 까지 내부 함수를 실행하지 않고 대기한다는 겁니다. for문은 속도가 아주 빠르므로 1초보다 훨씬 짧은 시간에 다음 i로 넘어가게 되고 또 다시 setTimeout을 실행합니다. 이렇게 아주 짧은 시간에 0~3까지 4번의 setTimeout이 실행됩니다. for문이 끝났을 때의 실행컨텍스트는 아래와 같습니다. GO( i=4 ) for 컨텍스트 ( empty ) setTimeout 컨텍스트 ( empty ) setTimeout 컨텍스트 ( empty ) setTimeout 컨텍스트 ( empty ) setTimeout 컨텍스트 ( empty ) 자, 이제 1초가 지났습니다. 3,4,5,6의 setTimeout 함수는 i 프로퍼티를 내부에 가지고 있지 않습니다. 따라서 상위에 있는 for 컨텍스트를 바라보게 되죠. for는 i를 4로 저장하고 있네요. 결과는 어떻게 될까요? 당연히 4를 거의 동시에 4번 출력하게 됩니다. 이제 문제를 해결해볼까요?? 중요한 점은 setTimeout 함수가 참조하는 i가 1초 후에도 변함없이 유지 되어야한다는 겁니다. for 컨텍스트는 i를 빠르게 변경하므로 i를 유지할 수가 없죠. 이것은 즉시실행함수(IIFE)를 이용하면 해결할 수 있습니다. 즉시실행함수는 함수를 선언과 동시에 실행하는 것을 말합니다. 예시를 보죠. (x => {   console.log(x) })(5) 위 결과는 5를 출력합니다. 익명함수 x => {...}를 ( )로 감싸고 곧바로 인자(5)를 넘겨주면 내부 함수가 실행되는 원리입니다. 이제 실행컨텍스트를 살펴볼까요? GO ( empty ) 즉시실행함수 컨텍스트 ( x = 5 ) console.log는 x를 참조합니다. x는 즉시실행함수에 저장되어 있네요. 즉 console.log는 5를 출력하게 됩니다. 감이 좀 오시나요? 즉시 실행함수는 새로운 컨텍스트를 만들어주고, 인자로 받은 값을 컨텍스트에 저장할 수 있습니다. 바로 이 점을 활용하는거죠. [솔루션] 아래가 바로 정답 코드입니다. for(var i=0; i<4; i++){   (x => {    setTimeout(() => {      console.log(x)   },1000*x)  })(i) }​ 1초 후의 실행컨텍스트를 살펴봅시다. GO( i=4 ) for 컨텍스트 ( empty ) 즉시실행함수 컨텍스트 ( x=0 ) setTimeout 컨텍스트 ( empty ) 즉시실행함수 컨텍스트 ( x=1 ) setTimeout 컨텍스트 ( empty ) 즉시실행함수 컨텍스트 ( x=2 ) setTimeout 컨텍스트 ( empty ) 즉시실행함수 컨텍스트 ( x=3 ) setTimeout 컨텍스트 ( empty ) 자, 어떤가요? 1초가 지난 후에 setTimeout이 실행되어도 x가 저장되어있으므로 x초마다 x를 출력할 수 있겠지요? 긴 글 읽어주셔서 감사합니다. 😍 추가) for문에서 let으로 만든 변수는 반복될 때마다 새로운 실행컨텍스트를 생성합니다. 따라서 var 변수를 let으로 변경하기만 해도 i값을 유지할 수 있답니다. ^^ 아래 코드로 간단하게 해결이 되네요! for(let i=0; i<4; i++){   setTimeout(() => {     console.log(i) },1000*i) }​ GO( empty ) for 컨텍스트 ( i = 0) setTimeout 컨텍스트 ( empty ) for 컨텍스트 ( i = 1) setTimeout 컨텍스트 ( empty ) for 컨텍스트 ( i = 2) setTimeout 컨텍스트 ( empty ) for 컨텍스트 ( i = 3) setTimeout 컨텍스트 ( empty ) 참고) 같은 비동기 함수인 setInterval()을 이용하는 방법도 있어요. 취향대로 선택하셔서 사용하세요. ^^; // setInterval() 사용 let i = 0 console.log(i++) let interval = setInterval(() => {  console.log(i++)  if(i >= 4) clearInterval(interval) }, 1000)​

3.어몽어스가 참 재미있어요 :D (어몽어스 설명, 미션, 규칙, 룰, 방법, 임포스터, 팁)
어몽어스는 미션이 있는 마피아 게임이에요 최대 10명이서 게임을 할수 있고 게임을 시작하면 크루원 vs 임포스터로 진영이 갈려요 크루원은 우주선 내에있는 미션들을 수행하면 왼쪽 위에 미션 게이지가 올라가고 그 미션 게이지가 꽉 차면 크루원의 승리. 임포스터는 크루원이 미션을 모두 끝내기 전에 크루원을 죽여서 임포스터 수와 일치시키면 임포스터의 승리. 임포스터는 킬을 들키지 않기 위해 방해공작을 펼칠 수 있답니다. 위 이미지는 시작 시에 처음 위치해 있는 식당이에요. 식당 한가운데는 Emergency 버튼이 있어서 이걸 누르면 긴급 회의가 시작되요. 긴급 회의에서는 임포스터를 찾아내는 투표를 진행할 수 있어요. 투표는 한 명을 선택하거나, 스킵하거나, 아무 것도 안할 수 있어요. 한 명이 가장 많이 선택되면 그 사람은 회의가 끝나고 우주선 밖으로 추방된답니다 ㅠㅠ 추방된 사람이 임포스터이면 게임은 크루원들의 승리로 끝나고, 임포스터가 아니면 게임은 계속 진행되요. 또 위에 보시면 지도에 노란색 느낌표가 있어요. 이건 크루원들이 수행해야하는 미션을 나타내는데요, 모든 크루원이 미션을 완료하면 크루원의 승리로 끝나게 된답니다. 임포스터에게 죽거나 추방되면 유령이 되는데 유령도 미션을 완료해야 미션이 모두 끝나게 되요. 죽어서도 미션을 게을리하면 안된답니다. ㅎㅎ 위 이미지는 미션이 가능할 때 나타나는 노란색 표시에요. 노란색 근처로 가게되면 오른쪽 아래의 USE 버튼이 활성화되고 이때 USE 버튼을 누르면 미션을 수행할 수 있답니다. 미션의 종류는 맵에 따라서 다양하니까 많이 해보셔야해요 ㅎㅎ 이제 임포스터의 역할에 대해 설명할건데요. 이건 사보타지 버튼을 눌렀을 때 나타나는 화면입니다. 임포스터는 사보타지를 통해 우주선을 고장내거나, 문을 차단하여 몰래 크루원들을 죽여서 승리할 수 있어요. 위의 X가 있는 동그라미 버튼은 해당 방의 문을 닫을 수 있는 버튼이에요. 문은 10초 동안 닫혀 있고, 각각 30초의 쿨타임이 적용된답니다. [원자로] 버튼을 누르면 경고음이 들리면서 30초의 제한시간과 원자로실의 위아래 미션이 활성화 되는데요. 제한시간 안에 원자로 미션을 완료하지 못하면 우주선이 고장나면서 임포스터의 승리로 끝나게 된답니다. 원자로 미션은 두 명이 위 아래로 나뉘어 손바닥 화면을 터치하고 있으면 쉽게 해제된답니다. 3.[React 정리] 1장) 초기 환경 세팅하기, create-react-app으로 앱 만들기
안녕하세요 임기입니다 :D React로 프로젝트를 하면서 정리를 할 필요가 있다고 생각이 들어서 글을 써봅니다. React 앱을 만들기 위한 강의를 시작해보겠습니다~ 저는 React를 처음 공부할 때 create-react-app을 이용했는데요. 이후에도 직접적인 바벨 설정을 건드릴 필요가 없을 경우 계속해서 CRA로 앱을 만들고 있습니다. CRA는 노드가 설치되어있으면 바로 실행할 수 있어요. 노드가 설치되어있지 않다면 node.js를 설치해줍니다. nodejs 홈페이지로 접속해서 안정적인 버전으로 설치하세요. 어렵지 않게 설치하실 수 있을거에요 :D 다음은 cmd나 git-bash를 실행합니다. 저는 리눅스 명령어들이 더 친숙해서 bash를 더 자주 사용합니다. 배쉬를 실행하셨다면 앱을 만들어 볼까요? npx create-react-app [프로젝트명]을 입력해보세요. 좀 기다리시면 진행상황이 보이며 몇 분 내로 다운로드가 끝나게 될거에요. 자! 다운로드가 끝났으면 디렉토리로 들어가 볼까요? 저는 프로젝트명을 example로 만들었기 때문에 cd example 을 입력합니다. 다음으로 code . 을 입력합니다. code . 은 현재 디렉토리를 visual studio code로 실행해줍니다. 참고로 explorer . 도 있는데요 이건 탐색기로 현재 디렉토리를 열어줍니다. 짜잔, vsc가 실행되셨나요? vsc는 Microsoft사에서 제작한 오픈소스 에디터랍니다. 전 세계에서 가장 많이 사용하는 에디터죠. vsc를 사용하고 있지 않으시다면 이참에 사용해보는건 어떨까요? ㅎㅎ 이제 본론으로 넘어와서 vsc왼쪽 상단의 문서처럼 생긴 버튼을 눌러서 Explorer를 열고 public 폴더의 index.html을 클릭해 봅시다. <!DOCTYPE html> <html lang="en">   <head>     <meta charset="utf-8" />     <link rel="icon" href="%PUBLIC_URL%/favicon.ico" />     <meta name="viewport" content="width=device-width, initial-scale=1" />     <meta name="theme-color" content="#000000" />     <meta       name="description"       content="Web site created using create-react-app"     />     <link rel="apple-touch-icon" href="%PUBLIC_URL%/logo192.png" />     <link rel="manifest" href="%PUBLIC_URL%/manifest.json" />     <title>React App</title>   </head>   <body>     <noscript>You need to enable JavaScript to run this app.</noscript>     <div id="root"></div>   </body> </html> 간단한 html파일이 열리는데요. 이 index.html이 리액트 앱에서 실행되는 단 하나의 페이지입니다. 리액트앱을 Single Page Application이라고 하죠? 여기서 싱글페이지가 이 index 파일을 의미한답니다. 하나의 페이지에서 js로 렌더링하여 화면을 바꾸는 것이죠. 그래서 전통적인 웹페이지와 다른 특수한 장치들이 필요하답니다. 그건 조금 후에 알아볼게요. index파일을 간단하게 설명해볼게요 <head> 태그는 리액트앱의 기본설정들이에요. meta charset : 페이지 인코딩 설정 link rel="icon" : 즐겨찾기 아이콘 이미지 설정 meta name="viewport" : 디바이스에 따른 화면크기 설정, 스케일 설정 meta name="theme-color" : 테마컬러 설정. 여기서는 검은색으로 설정했네요 meta name="description" : 페이지 내용 소개(검색 사이트 노출시킬 때 필요) link rel="apple-touch-icon" : 애플터치 아이콘 이미지 설정 link rel="manifest" : 모바일 앱 설정. 모바일 앱을 만들 때 각종 설정파일을 manifest에 작성해주는데 아이콘 이미지, 앱 이름 등을 설정해주는 부분이랍니다. title : 페이지 제목 설정 <body> 태그에는 <noscript>와 <div id="root"></div>밖에 없습니다. <body>     <noscript>You need to enable JavaScript to run this app.</noscript>     <div id="root"></div>   </body> <noscript>는 스크립트를 실행할 수 없는 환경에서 나타나는 문구입니다. 여기서 중요한건 바로 <div>인데요. 이 root div에 리액트 앱을 렌더링 한다는 걸 꼭 기억하세요 !! 글이 꽤 길어졌네요. ㅎㅎ 다음 강의에서 계속 할게요~ 다음 강의는 리액트앱의 구조를 살표보고 앱을 실행해봅시다! 그럼 이만 :D

4.귀여운 React Native 앱 Catbook!
안냥하세요 임기입니다 :D 오늘은 React Native로 만든 Catbook을 소개하려고 글을 썼어요. Catbook은 귀여운 고양이와 강아지 이미지를 볼 수 있는 간단한 네이티브 앱이랍니다. 외부 사이트의 api 정보를 가져오는걸 가공해서 보여준답니다. 간단한 구글 번역도 제공해요 ㅎㅎ 들어오셔서 구경해보세요~ web앱, Play store 그런데 React Native가 뭐냐고요? React Native는 facebook에서 제작한 프론트엔드 라이브러리로 React와 유사한 방식으로 모바일 native앱을 만드는 프로그램이랍니다. React 코드를 작성할 줄 알면 아주 쉽게 RN 앱도 만들 수 있어요 RN의 가장 큰 장점은 크로스플랫폼을 지원한다는 겁니다. 하나의 코드로 안드로이드, ios, web 앱을 동시에 만들수 있어요! 정말 놀랍죠? RN를 사용하지 않으면 똑같이 동작하는 프로그램을 만들기 위해서 세가지 언어로 세번 코딩을 해야하는 끔찍한 일이 벌어진답니다. RN는 이 외에도 정말 많은 장점이 있답니다 :D RN에는 정말 편리한 기능인 hot-reloading이 있어요. 네이티브 앱의 경우 코드를 수정하면 코드를 새로 컴파일하고 그걸 에뮬레이터에 올리고 테스팅 하는데 시간이 꽤 걸려요. 그래서 개발을 하다보면 흐름이 끊겨 답답하고 컴파일 하는데만 수 시간을 소모하곤 한답니다. 하지만 hot-reloading은 state를 초기화하지 않고 수정한 코드를 바로 화면에 적용시켜주어서 테스팅 시간을 획기적으로 줄일 수 있답니다. live-reloading은 소스코드 수정 시 앱 전체를 새로고침 하는건데요. 설정을 통해서 선택해주시면 됩니다. 그리고 RN을 얘기하다보면 빼놓을 수없는게 있는데요. 바로 expo입니다. RN을 처음 세팅하려면 꽤나 어렵고 복잡한 설정들을 잡아줘야합니다. 웹개발자에게 이런 세팅은 상당히 번거로운 일이죠. 이를 간편하게 세팅해주는게 바로 expo입니다. expo는 이외에도 아주 강력한 기능을 지원해줍니다. 바로 expo app을 통해 내 스마트폰에서 실시간으로 디버깅을 할 수 있는건데요. 정말 놀라운건 윈도우 환경에서 만든 앱을 바로 아이폰에서 테스팅 할 수 있습니다.. 언빌리버블.. 정말 expo 팀에게 찬사를 아끼지 않을 수 없네요. 또 아주 훌륭한 기능이 있는데요. 바로 expo publish입니다. 네이티브 앱은 구글플레이나 앱스토어에 앱을 게시해야 다른 사람들이 앱을 설치하고 사용할 수 있는데요. 앱을 게시하고 적용되는데 꽤 시간이 소요됩니다. 길게는 일주일 이상 수정사항을 적용하지 못하는 경우가 생길 수 도 있습니다. 자그마한 수정사항이라도 내가 원할 때에 앱을 게시 할 수 없다는건 아주 큰 리스크가 될 수 있죠. 하지만 expo는 실시간 publishing을 지원합니다! 내가 앱을 수정하고 expo publish를 하면 수정사항이 곧바로 앱에 적용됩니다. 정말 놀랍지 않을 수가 없습니다... 여기서 끝이 아닙니다. expo는 RN에서 다루지 않는 다양한 기능들을 지원합니다. 예를 들어 gps의 경우 RN에서 제공하는 기능은 몇 가지 뿐이지만 expo는 수 십 가지의 기능을 지원합니다. expo를 활용하시면 개발에 큰 이점을 줄 수 있답니다. 여러분들도 expo를 이용해서 간단한 네이티브 앱을 만들어보시면 어떨까요 ^^?

5.CSS 꿀팁
안녕하세요 임기입니다 :D CSS 꿀팁을 몇 가지 적어보고자 글을 씁니다. 저는 프로젝트를 시작하게 되면 가장 먼저 추가하는 스타일이 있는데요 바로 box-sizing 스타일입니다. * { box-sizing: border-box; } 박스사이징을 설명하기 전에 일반적인 상황을 먼저 보여드릴게요. 박스사이징을 설정하지 않은 상태에서 width : 100px인 버튼을 만들고 싶다면 아래와 같이 설정하게 됩니다. button { width: 100px; padding: 8px; border: 1px solid black; } 그런데 이렇게 되어있다면 실제로 button의 width는 100 + 8*2 + 1*2 = 118px이 됩니다. 저희가 원하는 크기는 100px인데 훨씬 커지므로 레이아웃이 깨져서 width를 82px로 줄여줘야하는 불편함이 있죠 :( 이런 불편함을 해결할 수 있는게 box-sizing 스타일입니다. 박스사이징을 설정하게 되면 border, padding, content(width, height) 요소가 content 사이즈를 넘어가지 않게 됩니다. 즉, width: 100px을 설정하면 자동으로 border와 padding을 감안해서 width가 82px인 것 처럼 보이게 되는 것이죠. 박스사이징은 width: 100% 처럼 퍼센티지 요소를 적용할 경우도 아주 잘 작동하게 됩니다. 단, 조심할 점이 하나 있는데요. margin은 width 밖에 적용되므로 margin을 적용해야하는 경우는 부모 엘리먼트의 전체 너비를 넘지 않도록 사이즈 조정을 해주어야 레이아웃이 깨지지 않습니다. 저는 이럴 때 CSS 함수인 calc()를 주로 사용합니다. 예를 들어서 menu와 content를 좌우로 배치하는 경우를 생각해봅시다. <!-- example.html --> <body> <div id="menu"/> <div id="content"/> </body> /* example.css */ #menu{ width: 300px; } #content{ width: 100%; } 위와 같이 스타일을 설정하면 menu와 content의 너비의 합이 100% + 300px이 되어서 레이아웃이 깨지게 됩니다. 그래서 이 두개를 같이 정렬하기 위해서는 width를 계산해서 입력해줘야합니다. 아래와 같이 말이죠. /* example.css */ #menu{ width: 300px; } #content{ width: calc(100% - 300px); } 주의 해야할 점은 calc() 안에 사칙연산 좌우로 띄어쓰기를 해주어야 잘 작동합니다. 띄어쓰기 없이 붙여쓰게 되면 CSS가 동작하질 않아요. 세번 째 꿀팁은 display: flex 입니다 :D flex는 요소들을 좌우 또는 상하 배치할 때 요긴하게 사용됩니다. 아래 예시를 볼까요? <!-- example.html --> <div class="flexDiv"> <button>1번</button> <button>2번</button> <button>3번</button> <button>4번</button> </div> /* flex.css */ .flexDiv{ background: lightblue; width: 100%; height: 100vh; display: flex; justify-content: space-between; align-items: flex-start; } button{ background: green; color: white; } 위 스타일을 적용하면 아래 그림 처럼 버튼이 배치되게 됩니다. .flexDiv 안에 버튼들이 나란히 위치해 있죠? 이걸 가능하게 해주는 속성이 justify-content입니다. justify-content에는 몇 가지 값들이 있는데 이걸 알아두셔야 상황에 맞는 스타일을 사용하실 수 있습니다. justify-content는 기본 축(default: row) 방향으로 요소들을 배치합니다. 기본 축이 가로이므로 한 줄에 여러개 요소들이 배치되는 모습이죠. space-between : 좌우 끝에 요소를 하나 씩 배치하고 남은 요소들을 가운데에 균일하게 배치합니다. 보통 페이지의 헤더 부분에 많이 사용하는 방식이죠. 위에 있는 그림처럼 배치됩니다. space-around : 모든 요소를 균일하게 배치합니다. 아래와 같은 모양이죠 flex-start : 축의 시작(왼쪽)에 요소들을 배치합니다. 딱히 flex를 쓸 필요가 없겠죠? flex-end : 축의 끝(오른쪽)에 요소들을 배치합니다. center : 축의 가운데에 요소들을 배치합니다. 다음으로 알아볼 속성은 align-items입니다. align-items는 justify-content와 반대 축을 기본 축으로 가집니다. justify-content가 가로 축이었다면 align-items는 세로 축이 되겠죠? 이번에는 버튼의 높이를 다르게 주어서 테스트 해봅시다. flex-start는 위쪽으로 정렬됩니다. 아래 이미지와 같습니다. flex-end는 아래쪽으로 정렬됩니다. 아래 이미지와 같습니다. center는 중앙으로 정렬됩니다. 저는 center나 flex-end를 가장 많이 사용합니다. 글과 이미지를 같은 높이로 맞추고 싶을 때 center가 유용하게 사용됩니다. 아래 이미지와 같습니다. 어떤가요 요소들을 편리하게 배치할 수 있겠죠? flex는 이외에도 축 방향 설정, 줄바꿈 등 몇 가지 설정을 더 할 수 있습니다. 더 고급 정보는 추후에 포스트할게요~ 이상입니다 :D
